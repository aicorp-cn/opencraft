## 报告漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请负责任地披露。

### 如何报告

**请勿在公开的 Issue 中报告安全漏洞。**

相反，请通过以下方式私下报告：

1. **GitHub 安全公告**（推荐）
   - 访问 [项目安全页面](https://github.com/aicorp-cn/opencraft/security/advisories)
   - 点击 "Report a vulnerability"
   - 填写漏洞详情

2. **邮件联系**
   - 发送邮件至项目维护团队
   - 邮件标题格式：`[SECURITY] OpenCraft 漏洞报告`

### 报告内容

请包含以下信息：

- 漏洞类型（如 XSS、注入、数据泄露等）
- 受影响的版本和组件
- 复现步骤
- 概念验证代码（如有）
- 可能的影响范围
- 建议的修复方案（如有）

### 响应时间

- **确认收到**：24 小时内
- **初步评估**：3 个工作日内
- **修复发布**：根据严重程度，通常在 7-14 天内

### 漏洞处理流程

1. 收到报告后确认并分配优先级
2. 验证漏洞并评估影响范围
3. 开发修复方案
4. 测试修复方案
5. 发布安全更新
6. 公开披露（如适用）

## 安全最佳实践

### API 密钥保护

本项目使用 LLM API 密钥，请务必：

- ✅ 使用环境变量存储 API 密钥
- ✅ 不要将 `.env` 文件提交到版本控制
- ✅ 在生产环境使用安全的密钥管理方案
- ❌ 不要在代码中硬编码 API 密钥
- ❌ 不要在公开的截图或日志中暴露密钥

### 生产部署建议

1. **启用 HTTPS**：确保所有通信加密
2. **配置 CORS**：限制允许的来源
3. **启用速率限制**：已内置 Slowapi 限流
4. **定期更新依赖**：检查并更新有漏洞的依赖
5. **监控日志**：记录并监控异常请求

### 依赖安全

我们建议使用以下工具检查依赖安全：

```bash
# 前端依赖检查
cd frontend-react
npm audit

# Python 依赖检查
cd server-python
pip-audit
```
